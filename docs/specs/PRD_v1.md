## 1. Executive Summary

「国試ノート」を**問題検索ツール**から**学習コンパニオン**に転換する。ファウンダー自身の国試浪人→合格体験から抽出した「発見→まとめ→暗記→確認」の学習サイクルをアプリが自動で回し、「アプリを開いたら今日やるべきことが2タップで始まる」体験を実現する。競合Emeryが「データベース検索型」で止まっている市場の空白を狙い、「上級者向け×コンパニオン型」のポジションを確立する。

---

## 2. Background & Context

### 市場の状況
- 薬剤師国試対策アプリ市場はEmeryの一強（4.8星/437件）
- 全アプリが「過去問を検索して解く」ツール型UX
- 「何を・どの順番で・どうやって勉強すべきか」を導くアプリはゼロ

### ファウンダーのドメイン知識
- 国試浪人→合格の実体験
- 1,200人のLINEオープンチャットコミュニティ運営（5-6年分のQ&Aデータ）
- 受験アンケート500件、就活アンケート1,000件+
- 自作の付箋ノート・暗記カード（Quizlet）

### 現在の状況
- MVP完了：11年分3,470問+AI解説、演習・暗記カード・付箋・分析ダッシュボード
- 問題マッピング（科目→分野→中分類→タグ）構築中
- UIがAnt Designデフォルトのまま、競合と差別化できていない

### なぜ今か
- Emeryの弱点（データ消失、解説品質、UX）が明確
- 問題マッピングが揃いつつあり、分野単位の学習導線が実現可能に
- Capacitorによるネイティブ化の技術的障壁が低い

## 3. Objectives & Success Metrics

### プロダクトビジョン

> **「次に何を勉強すべきか、もう迷わない」**
>
> 3,470問の過去問を、あなたの弱点に合わせて最適な順番で出題。
> 問題→付箋→暗記カード→再演習の学習サイクルを、アプリが自動で回す。

### デザイン原則

| # | 原則 | 意味 | 違反例 |
|---|------|------|--------|
| 1 | **2タップで演習開始** | ホーム→今日のメニュー→タップで学習が始まる | フィルターを3つ設定しないと始められない |
| 2 | **分野でマスターする** | 科目単位ではなく分野単位（薬理>抗がん剤）で進捗管理 | 「薬理の正答率70%」だけ表示 |
| 3 | **問題→付箋→暗記カードは1つの流れ** | 間違えた→まとめる→覚える→使える、がシームレス | 付箋と暗記カードが別タブで断絶 |
| 4 | **隙間時間ファースト** | 1-5分で意味のある学習ができる | 10問セット以外の選択肢がない |
| 5 | **データは絶対消えない** | ローカル優先+バックアップで信頼を築く | サーバー依存でオフラインNG |

### Goals（やること）

1. 学習コンパニオン型UXへの全面リデザイン
2. 分野単位のマスター進捗と学習導線の構築
3. 学習サイクル（問題→付箋→暗記カード→再演習）の自動化
4. Capacitorによるネイティブ化（プッシュ通知）
5. ファウンダーアセット（付箋・暗記カード・アンケート）のアプリ組み込み

### Non-Goals（やらないこと）

1. ❌ ランキング/ソーシャル機能（ユーザー数が少ない段階では逆効果）
2. ❌ 手書きメモ（バグの温床。テキスト付箋で十分）
3. ❌ デイリーチャレンジの固定3問形式（今日のメニューが上位互換）
4. ❌ 独自問題の作成（過去問+AI解説に集中）
5. ❌ Q&Aフォーラム/コミュニティ機能（Phase 3以降で検討）

### Success Metrics

| Metric | 現在（MVP） | Phase 1 目標 | Phase 2 目標 | 測定方法 |
|--------|-----------|-------------|-------------|---------|
| 初回→2日目の継続率 | 未計測 | 40% | 60% | Supabase analytics |
| 週間アクティブ率 | 未計測 | 30% | 50% | 7日以内にログイン |
| 1セッション平均問題数 | 未計測 | 8問 | 12問 | localStorage集計 |
| App Store評価 | - | 4.5以上 | 4.7以上 | ストアレビュー |
| 付箋→暗記カード転換率 | 0%（未連携） | 20% | 40% | 付箋から暗記カード生成数 |

## 6. 学習サイクル設計

### 公式付箋モデル

**重要な設計判断**: 付箋はユーザーが作るものではなく、**アプリが提供する公式コンテンツ**とする。

理由:
- 付箋の本質は「作用機序のフロー図」「ゴロ合わせ」「思考の整理」であり、手書きの視覚的表現が重要
- ユーザーがアプリ内でこの品質の付箋を作るのはハードルが高すぎる
- ファウンダー+コミュニティ協力者の付箋ノート約1,000枚（権利確認済み）が既にある
- 公式付箋は**課金コンテンツの候補**にもなる

### コアフロー：「発見→理解→暗記→確認」

```
                    ┌──────────────┐
                    │   ① 発見      │
                    │  問題を解く    │
                    │  穴が見つかる  │
                    └───────┬──────┘
                            │ 不正解
                            ▼
                    ┌──────────────────────┐
                    │   ② 理解              │
                    │  AI解説を読む          │
                    │        +               │  ← 新しい体験
                    │  📌 公式付箋が自動表示  │  ← 手書きフロー図・ゴロ
                    │  「この知識を覚えよう」  │     合わせ・覚え方
                    └───────┬──────────────┘
                            │ 付箋をブックマーク
                            ▼
                    ┌──────────────┐
                    │  ③ 暗記       │
                    │  暗記カード    │  ← 公式付箋に紐づく暗記カード
                    │  で反復       │  ← 隙間時間（歩き・電車・歯磨き）
                    └───────┬──────┘
                            │ カード学習後
                            ▼
                    ┌──────────────┐
                    │   ④ 確認      │
                    │  関連問題で    │  ← 同じ付箋を使う別の問題
                    │  使えるか検証  │
                    └───────┬──────┘
                            │
                   ┌────────┴────────┐
                   │                  │
                  正解              不正解
                   │                  │
                   ▼                  ▼
            マスター進捗UP      ①に戻る（次の穴）
            付箋の重要度UP
```

### 旧モデルとの違い

| 項目 | v1（旧） | v1.1（修正） |
|------|---------|-------------|
| 付箋の作成者 | ユーザーが自分で作る | **ファウンダーが公式で用意** |
| 付箋の形式 | テキスト入力 | **手書き画像 + AIテキスト要約** |
| ユーザーの操作 | 「付箋を作る」 | **「付箋を見る・ブックマークする」** |
| 表示タイミング | 解説画面に作成ボタン | **解説画面に関連付箋を自動表示** |
| 暗記カード | ユーザーが手動作成 | **公式付箋に紐づく暗記カードを事前生成** |
| 課金ポイント | なし | **公式付箋セットを課金対象にできる** |

### 各ステップの画面対応

| ステップ | 画面 | 主要アクション | 新規/既存 |
|---------|------|-------------|----------|
| ① 発見 | QuestionPage | 問題に回答して正誤判定 | 既存（UIリデザイン） |
| ② 理解 | QuestionPage 解説部 | AI解説 + 関連する公式付箋を自動表示 | **拡張** |
| ② ブックマーク | QuestionPage | 公式付箋を「マイ付箋」にブックマーク | **新規** |
| ③ 暗記 | FlashCardPage | 付箋に紐づく暗記カードで反復学習 | 既存（データ連携強化） |
| ④ 確認 | PracticePage→QuestionPage | 「この付箋の関連問題を解く」 | **新規** |

### 公式付箋のデータモデル

```typescript
interface OfficialNote {
  id: string
  title: string              // 「交感神経 α1受容体の作用機序」
  imageUrl: string           // 手書きノート画像（PNG/WebP）
  textSummary: string        // AIで生成したテキスト要約（検索・暗記カード用）
  subject: QuestionSubject   // 「薬理」
  field: string              // 中分類「交感神経系」
  tags: string[]             // 「α受容体」「血管収縮」
  linkedQuestionIds: string[] // この知識が必要な問題ID群
  linkedCardIds: string[]    // 紐づく暗記カードID群
  importance: number         // 紐づく問題数から自動算出
  tier: 'free' | 'premium'  // 課金区分
}

interface BookmarkedNote {
  noteId: string             // OfficialNoteのID
  userId: string
  bookmarkedAt: string
  reviewCount: number        // 何回見たか
}
```

### 公式付箋の制作パイプライン

```
手書きノート（紙）
    │
    ▼ スキャン / 撮影
    │
PNG画像（約1,000枚）
    │
    ├──→ AI画像認識 → テキスト要約を自動生成
    │
    ├──→ 科目・分野・タグの手動分類
    │
    ├──→ 問題IDとの紐づけ（問題マッピングを利用）
    │
    └──→ 暗記カードの事前生成（テキスト要約をfront/backに）
```

### 7.3 問題解答画面 — リデザイン優先度: ★★☆

| 機能 | 現在 | v2.0 | 優先度 |

#### 課金設計（全体）

Emeryの課金は「広告消し+便利機能」（機能課金）。国試ノートは**コンテンツ課金 + AI機能課金**のハイブリッドで差別化。

##### 3つの価値レイヤー

| レイヤー | 内容 | モデル | 価格帯 | Phase |
|---------|------|--------|--------|-------|
| **🆓 Free** | 過去問3,470問+AI解説（全部無料）、演習・フィルター・分析、公式付箋200枚、暗記カード（Free付箋分）、オンボーディング実力測定 | 無料 | 0円 | Phase 1 |
| **💎 付箋パック** | 全公式付箋1,000枚 + 全暗記カード。noteやメルカリで売れている付箋まとめノートと同等の価値 | **買い切り** | **2,980〜4,980円** | Phase 2 |
| **🤖 AIパック** | AI生成オリジナル類題（分野別）+ 先輩コーチAI（RAGアドバイス）。毎月新問題追加 | **月額** | **480〜780円/月** | Phase 3 |

##### 課金モデルの根拠

| 比較 | Emery | 国試ノート |
|------|-------|----------|
| 課金の軸 | 便利機能（広告消し・メモ）= 「不満の解消」 | **コンテンツ+AI** = 「合格に直結する価値」 |
| 月額 | 1,280円/月 | 480-780円/月（AI機能のみ） |
| 買い切り | なし | 2,980-4,980円（付箋パック）|
| 無料の範囲 | 過去問+解説 | 過去問+AI解説+付箋200枚 |

**買い切り推奨の理由**: 受験生は試験日が決まっており、月額は心理的抵抗あり。note/メルカリで付箋ノートが2,000-5,000円で売れている市場実績がある。

##### AI類題の体験イメージ（Phase 3）

```
問題を解く → 不正解（交感神経α1の問題）
  │
  ├── 過去問の類題（同じ分野）3問 ← 🆓 無料
  │
  └── AI生成の類題 5問            ← 🤖 AIパック
      「α1受容体の作用を問う新しい切り口」
      「臨床現場での判断を問うシナリオ型」
```

ファウンダーの体験：「過去問だけだと同じ問題の暗記になる。違う切り口で問われたときに解ける武器になっているかが重要」→ AI類題の価値。

##### 先輩コーチAI（Phase 3 — AIパックに含む）

受験アンケート500件をRAGソースとして、模試の点数や学習状況に応じたアドバイスを生成。

```
ユーザー: 「統一模試1回目140点。間に合う？」

先輩コーチAI:
  📊 過去の先輩データ（500件）から分析：
  統一模試1回目 130-150点だった先輩 82人中：
  ✅ 72人（88%）が最終的に合格

  合格した先輩の共通パターン：
  ・必須問題の正答率を先に90%以上にした（67人）
  ・苦手科目を2科目に絞って集中した（54人）

  💬 先輩の声：
  「140点から230点まで上げました」（109回合格・Aさん）

  🎯 あなたへのおすすめ：
  → 必須問題の苦手分野を今日のメニューで始めましょう
```

#### 付箋の重要度スコア

```
付箋の重要度 = 紐づく問題数 ×（1 + ユーザーの不正解率）

例：「交感神経α1受容体」の付箋
  → 12問に紐づく ×（1 + 不正解率60%）= スコア 19.2（高い）
  → 表示：🔥 よく出る！12問で使う知識
  → ユーザーごとに不正解率が違うので、重要度が個人化される
```

### 7.6 オンボーディング — 新規: ★★★

**コンセプト**: 「5分で今の実力がわかる → 最適な学習プランが自動設定される」

Duolingoのプレースメントテストと同じ発想。ただし語学の「レベル測定」ではなく、国試の「**どの分野に穴があるか**」を測定する。全員が同じゴール（合格点230点）を目指すので、穴の特定が重要。

#### フロー

```
画面1: ようこそ！
  「あなたの目標を教えてください」
  [🎓 国家試験対策（6年生）]  → 効率型プリセット
  [📚 CBT対策（4年生）]       → 入門型プリセット
  [🔄 再チャレンジ（既卒）]   → 分析型プリセット

画面2: 「今の実力を5分で測りましょう」
  各科目の頻出分野から必須問題を3問ずつ出題
  ・主要6科目 × 3問 = 18問（約5分想定）
  ・正答率70%以上の基本問題のみ使用
  ・サクサク回答（1問15-20秒想定）
  ・解説は表示しない（テンポ重視）
  ・進捗バー表示「3/18問」

画面3: 結果レポート
  「あなたの得意・苦手マップ」
   薬理 ●●○  67%
   化学 ●●●  100% ✅
   生物 ●○○  33%  ← ここから始めよう！
   物理 ●●○  67%
   薬剤 ○○○   0%  ← 要注意！
   実務 ●●●  100% ✅

  → 苦手分野が自動特定
  → 「今日のメニュー」に苦手分野の演習が即反映
  → 「まずは薬剤の基本から始めましょう！」

→ ホーム画面（パーソナライズ済み）
```
