# User Research Report: pharma-exam-ai（国試ノート）

**Date**: 2026-03-23
**Data sources**: 競合アプリレビュー（Emery App Store 437件）、市場比較記事、アプリ使い方ガイド、自アプリコード分析
**Confidence level**: 仮説ベース（ユーザーインタビュー未実施 → 要バリデーション）

---

## Executive Summary

薬剤師国試対策アプリ市場は、Emeryが一強（4.8星/437件）だが、**レビューの不満は「UIの操作性」「データ消失」「解説の読みにくさ」に集中**している。ユーザーは3つの明確なセグメントに分かれ、それぞれ異なる学習行動パターンを持つ。pharma-exam-aiは「AI解説」「オフライン」「データ安全性」で差別化可能だが、**最大の勝ち筋は「何を勉強すべきか迷わないUX」**にある。現状のEmeryを含む全アプリが「問題データベースの検索ツール」であり、「学習コンパニオン」として設計されたアプリは市場に存在しない。

---

## Personas

### Persona 1: 田中 美咲（22歳）— 「効率よく最短で受かりたい」

- **Who**: 薬学部6年生。研究室と就活と国試対策を並行中。OSCE・CBT済み。
- **Primary JTBD**: 通学時間や空きコマに、苦手分野を効率的に潰したい。電車の中でも片手で問題を解けるようにしたい。
- **Key pains**:
  1. 「何から手をつけていいかわからない」— 3,000問以上あって圧倒される
  2. 電車で片手操作だとフィルター選択が面倒（ドロップダウン何個も開く）
  3. 解説が長文で改行なし → 電車で読みきれない
- **Key gains**:
  - 「今日はこれだけやればOK」と言ってくれるアプリ
  - 苦手が数字で見える化される達成感
  - 友達より進んでいるか確認できる安心感
- **Behavioral pattern**:
  - 朝の通学電車（30分）+ 昼休み（20分）+ 寝る前（15分）
  - **短時間×高頻度**。1セッション10問が限界
  - 「全範囲ランダム」より「苦手に絞って繰り返す」スタイル
- **Prevalence**: 推定60%（最大ボリュームゾーン）
- **Key quote**: 「正答率低い問題ばっかり出してほしい。得意なのはもういい」

---

### Persona 2: 鈴木 拓也（24歳）— 「落ちたくない。不安を数字で潰したい」

- **Who**: 既卒受験者（1回目不合格）。薬局でバイトしながら再チャレンジ。
- **Primary JTBD**: 前回の失敗パターンを分析して、**同じ間違いを繰り返さない**勉強がしたい。限られた時間で効果的に学習したい。
- **Key pains**:
  1. 「どの分野が原因で落ちたか」を分析する方法がない
  2. 仕事の合間の隙間時間（30分）しかない → 時間効率命
  3. 有料アプリに月1,280円は厳しい（バイト生活）
- **Key gains**:
  - 科目別・単元別の弱点が可視化される
  - 「前回の模試で間違えた問題」だけリプレイできる
  - 無料で使える基本機能の充実
- **Behavioral pattern**:
  - バイト休憩（30分）+ 帰宅後（1-2時間集中）
  - **じっくり解説を読む** → 付箋やメモを多用
  - 過去問を年度単位で通しで解く「模試型」学習
- **Prevalence**: 推定20%（既卒受験者 = 約3,000-4,000人/年）
- **Key quote**: 「解説読んで理解しても、似た問題で同じ間違いする。パターンで教えてほしい」

---

### Persona 3: 佐藤 あかり（20歳）— 「CBTをまず乗り越えたい」

- **Who**: 薬学部4年生。CBT対策で初めて過去問アプリを使う。
- **Primary JTBD**: CBT必須問題の範囲に絞って、基礎を固めたい。国試はまだ先だが、早めに始めておきたい。
- **Key pains**:
  1. 国試向けアプリは情報量が多すぎて怖い（345問中の必須だけやりたい）
  2. 「実践問題」「連問」は今は必要ない → フィルターの概念がわからない
  3. 友達が使ってるからEmery入れたけど、設定がわからず放置
- **Key gains**:
  - 「CBT対策モード」のようなシンプルな入口
  - 基礎科目（物理・化学・生物）の理解度が見える
  - 難しすぎる問題を避けて自信をつけたい
- **Behavioral pattern**:
  - 講義の復習として使う（週2-3回、各30分）
  - 「必須問題」のみ → 年度横断で科目別に解く
  - **ゲーム感覚**（ストリーク、バッジ）でモチベ維持
- **Prevalence**: 推定20%（潜在市場 — 現状ほとんどの4年生はアプリ未使用）
- **Key quote**: 「難しい問題は飛ばしたい。まず基本を全部○にしたい」

## User Segments

| セグメント | サイズ | Primary JTBD | Product Fit | 課金意欲 | 成長性 |
|-----------|--------|-------------|-------------|---------|--------|
| **⚡ 効率型（美咲）** | 60% | 隙間時間で苦手を最短で潰す | ◎ プリセット + AI解説 | 中（無料→月500円） | 安定 |
| **📊 分析型（拓也）** | 20% | 弱点を可視化して戦略的に学ぶ | ◎ 分析ダッシュボード + 履歴 | 高（月1,000円OK） | 中 |

### 最重要セグメント

- **最大価値**: 📊 分析型 — 課金意欲が高く、LTV（生涯顧客価値）最大
- **最大成長**: 🌱 入門型 — 4年生→5年→6年と2年間使い続ける可能性。初期獲得がカギ

### pharma-exam-aiの最適ポジション

```
                課金意欲 高
                    │
         分析型（拓也）●   ← pharma-exam-aiのスイートスポット
                    │
    入門型（あかり）●───────● 効率型（美咲）
                    │
                課金意欲 低

    シンプル ─────────────────── 高機能
```

**戦略**: 効率型を入口にし（最大ボリューム）、分析型で課金転換、入門型で将来のパイプライン確保

## Customer Journey Map

### Phase 1: 認知（国試1年前〜6ヶ月前）

| 項目 | 詳細 |
|------|------|
| **タッチポイント** | 先輩の口コミ、薬学ゼミ、Twitter/X、大学の掲示板 |
| **ユーザーの感情** | 😰「そろそろ勉強始めないと...」不安 |
| **行動** | App Store で「薬剤師 国試」検索、友達に聞く |
| **ペインポイント** | アプリが多すぎてどれが良いかわからない |
| **機会** | **「3分で始められる」を訴求。インストール→最初の1問が30秒以内** |

### Phase 2: 試用（初日〜1週間）

| 項目 | 詳細 |
|------|------|
| **タッチポイント** | アプリ初回起動、ウェルカム画面、最初の演習 |
| **ユーザーの感情** | 🤔「使い方がわからない...」→ 😊「おっ、良さそう」or 😩「面倒、やめよ」 |
| **行動** | 適当にフィルター設定→10問解く→解説読む→続けるか判断 |
| **ペインポイント** | ⚠️**最大の離脱ポイント**。フィルターが複雑、何をすればいいか不明 |
| **機会** | **スマートプリセット「まずはこれ！」でワンタップ開始。迷わせない** |

### Phase 3: 習慣化（1週間〜1ヶ月）

| 項目 | 詳細 |
|------|------|
| **タッチポイント** | 毎日の通知、ストリーク表示、ホーム画面の「今日のメニュー」 |
| **ユーザーの感情** | 🔥「続けてる！」or 😐「飽きてきた」 |
| **行動** | 毎日10問ペースで解く、苦手科目を重点的にやる |
| **ペインポイント** | 進捗が実感しにくい、モチベーション維持 |
| **機会** | **単元マスター進捗バー（実装済み）+ ストリーク報酬強化** |

### Phase 4: 深化（1ヶ月〜3ヶ月）

| 項目 | 詳細 |
|------|------|
| **タッチポイント** | 分析ダッシュボード、苦手問題リスト、暗記カード |
| **ユーザーの感情** | 📊「データを見て戦略を立てたい」 |
| **行動** | 科目別正答率を確認、弱点分野を集中演習、ノート作成 |
| **ペインポイント** | 「次に何をすべきか」の判断を自分でしないといけない |
| **機会** | **AIが「次はこの単元をやるべき」をレコメンド** |

### Phase 5: 直前期（試験3ヶ月前〜当日）

| 項目 | 詳細 |
|------|------|
| **タッチポイント** | 模試結果、年度別演習、全範囲ランダム |
| **ユーザーの感情** | 😱「間に合うかな...」高ストレス |
| **行動** | 年度別に本番形式で通し演習、弱点の最終潰し |
| **ペインポイント** | 本番形式の模試がない、タイマーがない |
| **機会** | **模試モード（時間制限 + 本番同様の問題数配分）** |

### Phase 6: 合格後

| 項目 | 詳細 |
|------|------|
| **タッチポイント** | 合格報告、SNSシェア、口コミ |
| **ユーザーの感情** | 🎉「受かった！このアプリのおかげ」 |
| **行動** | 後輩に勧める、レビューを書く |
| **ペインポイント** | 合格後に使う理由がない |
| **機会** | **合格体験記シェア機能、後輩紹介プログラム** |

## ジャーニー上の最大の離脱ポイント（要対策）

```
認知 → [試用] → 習慣化 → 深化 → 直前期 → 合格
         ↑
     ここで70%が離脱（推定）

原因:
1. 「何をすればいいかわからない」
2. 「フィルター設定が面倒」
3. 「10問解いたけど、次どうすればいいか不明」
```

**→ 対策 = スマートプリセット + 今日のメニュー（実装済み）+ オンボーディング改善**

## Key Insights

### 1. エメリーの成功要因は「先行者利益」であり、UXではない

エメリーの4.8星は「他にまともなアプリがなかった」という文脈での評価。レビュー内の不満（データ消失、ナビゲーション、解説品質）は深刻だが、代替がないため高評価のまま。**UXで上回れば十分にスイッチ可能**。

### 2. ユーザーが本当に求めているのは「考えなくていい学習体験」

3,000問の中から条件を組み立てて検索する作業自体が認知負荷。Duolingoが語学学習で証明したように、「アプリを開く→今日やることが1画面に表示→タップして開始」が理想形。**pharma-exam-aiのHomePage「今日のメニュー」はこの方向で正しい。これを全面に押し出すべき。**

### 3. 「入門型ユーザー」が完全に無視されている

市場の全アプリが6年生/既卒を想定。4年生CBT層は「必須問題だけ解きたい」のに、全機能が見えてしまい圧倒される。**「CBTモード」ボタン1つで、必須問題 + 基礎科目に絞られるシンプルUIを提供すれば、この層を独占できる。**

### 4. 解説品質はpharma-exam-aiの最大の武器

エメリーの最大の弱点「解説に改行がない、読みにくい、誤字多い」に対して、pharma-exam-aiはAI生成の構造化された解説を持つ。**これは体験後すぐに差が実感できるポイント。**

### 5. オフライン対応は「あると嬉しい」ではなく「必須」

通学電車（地下鉄）での学習が最大のユースケース。PWA対応は大きなアドバンテージ。

## Recommendations

### 短期（今すぐ反映すべき）

1. **オンボーディングフローの追加**: 初回起動→ペルソナ選択（「国試対策」「CBT対策」「再チャレンジ」）→自動プリセット設定
2. **スマートプリセットの実装**（演習ページリデザイン案の通り）
3. **ホーム画面「今日のメニュー」をメイン導線に昇格**（現在はページ下部）

### 中期（1-2ヶ月以内）

4. **CBTモードの追加**: 必須問題のみ + 正答率80%以上に絞った「基礎固め」モード
5. **模試モードの追加**: 年度×区分で本番形式（制限時間付き）
6. **AI学習レコメンド**: 回答履歴から「次はこの単元をやるべき」を提案

### 長期（3ヶ月以降）

7. **ソーシャル機能**: 匿名の進捗ランキング（エメリーの成功要因）
8. **合格後の後輩紹介フロー**: 口コミドリブンの成長ループ

## Open Questions（要バリデーション）

1. **4年生CBTユーザーは実際にアプリを使うか？** → 薬学部4年生3-5人にヒアリング推奨
2. **既卒受験者の課金閾値はいくらか？** → 500円/月 vs 1,000円/月のWTP調査
3. **「今日のメニュー」の自動レコメンドは信頼されるか？** → A/Bテスト（プリセット vs 自分で選ぶ の利用率比較）
4. **ストリーク・バッジ等のゲーミフィケーションは薬学生に有効か？** → Duolingo型が合うか、AMBOSS型（知的・ストイック）が合うか
