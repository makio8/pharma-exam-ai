# 連問内テキスト崩れ問題の修復設計

## 問題

連問内の一部問題で、PDF抽出時のテキストが崩れている。特に数式・表・化学構造式を含む問題で顕著。

### 具体例
- r100-201: クラペイロン・クラウジウスの式 `dp/dT = pΔH/RT²` が `dp p H\n=\ndT RT` のようにバラバラに展開
- 約65問の連問メンバーでテキスト崩れの可能性

### 現在の状態
- PDFファイル: なし（抽出済みデータのみ）
- PDFツール: pdftoppm, pdftotext あり
- 既存画像: 909枚（public/images/questions/）
- question_text_original: 807問に存在（PDF生テキスト保持）

## 解決策の比較

### A. PDFを再取得してページ画像を追加
- **方法**: 厚労省サイトからPDF再ダウンロード → pdftoppm で該当ページを画像化 → image_url設定
- **メリット**: 数式・構造式が完全に正しく表示される
- **デメリット**: PDF再取得が必要（サイト変更リスク）、手動で問題番号→ページ番号のマッピングが必要
- **工数**: 中（PDF取得+画像抽出+データ更新）

### B. question_textのテキストクリーニング
- **方法**: 崩れたテキストパターンを正規表現でクリーニング（改行除去、空白正規化等）
- **メリット**: 外部依存なし、すぐ実行可能
- **デメリット**: 数式は完全には復元できない（dp/dT等）、構造式は不可能
- **工数**: 小

### C. 崩れた問題にimage_urlを追加（既存画像の再利用）
- **方法**: 該当問題の前後の問題にimage_urlがある場合、同じPDFページの画像を共有。または連問グループのimage_urlからページを推定
- **メリット**: 追加データ不要の可能性
- **デメリット**: 1問1ページとは限らない、画像が存在しない問題も多い
- **工数**: 小〜中

### D. ハイブリッド（推奨）
1. まずテキストクリーニング（B）で可能な範囲を改善
2. 数式/構造式を含む問題は、question_textに「※ 数式を含む問題です。PDFの画像でご確認ください」注記を追加
3. 将来のPDF再取得時にimage_urlを追加（A）

## 推奨: D（ハイブリッド）

**理由:**
- 即座に実行可能（PDFなしで対応）
- テキストクリーニングで大半の問題は改善される
- 数式問題は注記で学習者に伝える
- PDF再取得は別セッションで対応可能

## 対象ファイル
- Modify: `src/data/real-questions/exam-{100..111}.ts` — テキストクリーニング
- Create: `scripts/clean-question-text.ts` — クリーニングスクリプト

## テキストクリーニングルール
1. PDF改行の正規化: `\n` が単語の途中にある場合は除去
2. 余分な空白の圧縮: 3文字以上の連続スペース → 1スペース
3. 「問 XXX（科目）」ヘッダーの除去（連問内の各問題にはLinkedQuestionViewerのヘッダーがあるため重複）
4. 選択肢テキストの正規化（改行除去）

## 完了基準
- テキスト崩れが視覚的に改善される（完全な修正は不要）
- ビルド成功
- 既存E2Eテストが通る
