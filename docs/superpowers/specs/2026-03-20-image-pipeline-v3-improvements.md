# 画像パイプライン v3 改善設計スペック

**日付**: 2026-03-20
**ステータス**: ドラフト（次セッションで実装）
**前提**: v2 パイプライン完了済み（867問クロップ・テキストクリーニング・品質テスト）

## 背景：実機検証で発見された課題

v2パイプライン完了後、実機（iPhone）とエメリー（薬ゼミアプリ）との比較検証を実施。
8問を詳細に確認した結果、5カテゴリの課題を特定。

## 課題一覧

### 課題1: 画像の余白が大きい（🔴 高優先）

**現象**: ページ末尾の問題がページ下端までクロップされ、巨大な余白・ページ番号・トンボが含まれる
**該当例**: 103回問91（表の下に半ページ余白）、107回問197（トンボ・「−2−」まで表示）、100回問233（「−31−」が見える）
**原因**: `calcCropRegion` でページ末尾の問題は `imageHeight - 10` までクロップする設計
**解決策**: `sharp` の `trim()` で白い余白を自動除去 + ページ番号（「− N −」パターン）の領域をカット
**工数**: 小（crop-utils.ts の修正 + 全画像再クロップ）

### 課題2: choices空だがテキスト選択肢で解ける問題がある（🔴 高優先）

**現象**: 本来choicesにテキストを入れれば解ける問題が、choices空+画像表示になっている
**該当例**:
- 103回問91（表: 分子間相互作用 → 現象）
- 100回問233（表: 植物 → 有毒成分）
- 100回問239（テキスト選択肢がPDFでは取得できなかった）
**原因**: pdftotext -layout で選択肢抽出に失敗した問題が一律 choices空 になっている
**解決策**: AI（Claude API）で画像から選択肢テキストを抽出し、choices を復元
  - 対象: choices空 867問のうち、選択肢がテキストとして読める問題（表形式・リスト形式）
  - 構造式や図の選択肢（問6, 問8等）は画像のまま維持
  - 判定: AIに「この問題の選択肢はテキスト化可能か？」を判定させる
**工数**: 中（AIバッチ処理スクリプト、867問のうち推定200-300問が対象）

### 課題3: question_text のOCR品質が低い（🟡 中優先）

**現象**: pdftotext -layout 由来の改行崩れ、数字混入、不自然な分断
**該当例**:
- 107回問197: 「問 196−197　₁₇歳男性。身長₁₇₅cm」（下付き数字混入）
- 100回問2: 「A　B」「k = 0.01min」がそのまま残る
- 100回問8: 「1つ選」と「べ。」が分断
**原因**: pdftotext -layout の出力をそのまま使用。PDF内部の文字配置が原因
**解決策**: AIで画像から問題文テキストを正確に再生成
  - choices復元（課題2）と同時にAIで question_text も再生成
  - 問い文だけでなく、シナリオ部分も正確に抽出
**工数**: 課題2と同時実行可能

### 課題4: 連問（実践問題）の構造化（🟡 中優先）

**現象**: 実践の連問（問196-197、問232-233、問238-239等）が個別の問題として表示される
**理想**: エメリーのように共通シナリオ+処方画像を1回表示し、子問題を順番に解く
**該当例**: 107回問196-197（処方情報が2問に重複表示）、100回問238-239（共通シナリオ重複）
**解決策**:
  1. Question型に `linked_group?: string`（例: "r107-196-197"）と `linked_scenario?: string` を追加
  2. PDFの「問 196−197」パターンから連問グループを自動検出
  3. UIで連問グループを一括表示（共通シナリオ → 子問題1 → 子問題2）
**工数**: 大（データ構造変更 + UI変更 + 全年度の連問検出）

### 課題5: choicesあり問題の補助画像が欠如（🟡 中優先）

**現象**: choicesありの問題でも、処方箋・検査値表などの補助画像が必要な問題がある
**該当例**: 110回問286-287（処方情報がテキストだが、画像の方が見やすい）
**原因**: 現在image_urlはchoices空の問題にのみ設定。choicesありの問題は画像なし
**解決策**:
  - 連問のシナリオページ画像を共有画像として設定
  - 課題4（連問構造化）と同時に対応
**工数**: 課題4に含まれる

## 優先順位と推奨実装順序

| 順序 | 課題 | 理由 | 工数 |
|------|------|------|------|
| **1** | 画像余白トリム（課題1） | 即効性高い。sharp.trim()で簡単に改善 | 30分 |
| **2** | AI choices復元 + テキスト再生成（課題2+3） | 最も多くの問題を改善。867問→推定200-300問のchoices復元 | 2-3時間 |
| **3** | 連問構造化 + 補助画像（課題4+5） | UI変更が大きい。MVPとしては後回しでOK | 3-4時間 |

## 技術的アプローチの転換

### v2（現在）: pdftotext ベース
- ❌ OCR品質が低い（改行崩れ、数字混入）
- ❌ 表形式のchoicesが抽出できない
- ✅ 高速、API不要

### v3（推奨）: AI Vision ベース
- ✅ 画像から正確なテキスト抽出
- ✅ 選択肢のテキスト化判定が可能
- ⚠️ API コスト（Claude API or Gemini Flash）
- ⚠️ レート制限への対応が必要

## スコープ外

- 連問UIの完全実装（Phase 2以降）
- 画像の解像度最適化
- choices復元後の正答率追加
