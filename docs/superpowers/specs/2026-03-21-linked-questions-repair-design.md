# 連問データ修復 + UI改善 設計スペック

## 日付: 2026-03-21

## 背景と課題

薬剤師国試の実践問題（問196-325）は2〜4問の連問セットで出題される。共通シナリオ（患者情報・処方・検査値など）を読んでから各問を解く形式。

### 現状の問題

1. **linked_group 欠落**: 1,632問中1,124問(69%)に `linked_group` が未設定
2. **先頭問題の欠落**: 467グループ中443グループ(95%)で先頭問題が `linked_group` から漏れている
   - 例: 問196は連問セットの一部だが、`linked_group` なし。問197-199のみ `"r100-197-199"` が設定
3. **シナリオ不完全**: `linked_scenario` が1行要約のみで、共通シナリオ全文が入っていない
4. **ナビゲーション**: 1問ずつ移動するため、連問の途中に着地してしまう

### 影響範囲

- **実践問題**: 全11年度（100-110回）× 問196-325 = 約1,430問（主な修復対象）
- **理論問題の連問**: 一部の年度で理論セクションにも連問あり（例: 103回 問119-120、104回 問137-140 等）。`category` にグループ情報があるものは修復対象に含める
- 問326-345（実務セクション）はスタンドアロン → 修復対象外

## 設計

### 1. データ修復スクリプト

#### 1-1. linked_group の修復

**データソース優先順位:**
1. `category` フィールド: 先頭問題に「一般 実践問題 - 問 196,197,198,199」形式でグループ情報あり
2. 既存 `linked_group`: 先頭以外の問題に既に設定されているケースの補完
3. `question_text_original` 内のマーカー: 「問 206‑207」形式のヘッダーからグループ推定

**処理フロー:**
```
1. 全年度の全問を走査（理論問題の連問も対象）
2. category にグループ情報がある問題（先頭問題のみに存在）を検出
   - 「一般 実践問題 - 問 196,197,198,199」→ [196,197,198,199]
   - 「一般 理論問題 - 問 119,120」→ [119,120]
3. リスト内の全問に linked_group を "r{year}-{first}-{last}" 形式で設定
4. category が空だが既存 linked_group がある問題 → その範囲で補完
5. それでも推定不可 → question_text_original のマーカーで推定
6. 最終的に未解決 → ログ出力して手動対応
```

**重要: `category` は先頭問題にしか存在しない。** 先頭問題のcategoryをパースし、リスト内の全問に `linked_group` を設定する処理。

**linked_group 命名規則の変更:**
- 旧: `"r100-197-199"` （先頭問題を除外していた）
- 新: `"r100-196-199"` （先頭問題を含む正しい範囲）
- **既存の全 `linked_group` を新形式に一括書き換え**（旧形式は残さない）

#### 1-2. linked_scenario の修復（共通シナリオ全文）

**注意:** `split-linked-question-text.ts` で既に157問のテキスト分割が実行済み。分割済みの先頭問題からはシナリオ部分が除去されている可能性がある。

**抽出元の優先順位:**
1. `question_text_original`（PDF原文、最も正確）→ シナリオ部分を抽出
2. 先頭問題の `question_text`（未分割の場合のみシナリオを含む）
3. 既存の `linked_scenario`（要約のみでも無いよりマシ）

**シナリオ抽出ロジック:**
- `question_text_original` または `question_text` を「問XXX（科目）」マーカーで分割
- マーカーより前の部分 = 共通シナリオ
- マーカー以降 = その問固有の問題文
- 既に分割済み（マーカーなし）の場合 → 既存 `linked_scenario` を維持

#### 1-3. question_text のクリーニング

- 各問の `question_text` から共通シナリオ部分を除去
- 各問固有の問題文のみを残す
- 既に `split-linked-question-text.ts` で157問分を処理済み → 同じロジックを全問に適用

### 2. UI変更

#### 2-1. LinkedQuestionViewer の改善

**レイアウト（本番試験と同じ配置）:**
```
┌─ 🔗 連問（4問セット）─────────────┐
│                                    │
│  [共通シナリオ全文]                  │
│  患者情報・処方・検査値など          │
│                                    │
├─ 問196 実務 ──────────────────────┤
│  問題文（固有部分のみ）             │
│  ○ 選択肢1                        │
│  ○ 選択肢2                        │
│  ○ ...                            │
│  [回答する]                        │
│                                    │
├─ 問197 薬剤 ──────────────────────┤
│  問題文（固有部分のみ）             │
│  ○ 選択肢1                        │
│  ○ ...                            │
│  [回答する]                        │
│                                    │
├─ 問198 物化生 ────────────────────┤
│  ...同様...                        │
│                                    │
├─ 問199 物化生 ────────────────────┤
│  ...同様...                        │
│                                    │
└────────────────────────────────────┘
```

- 各問の問題文の直後に選択肢を配置（問題と選択肢が離れないように）
- 各問ごとに独立して回答・採点・解説表示
- 共通シナリオは最上部に1回だけ表示

#### 2-2. ナビゲーション（セット単位）

- 「次の問題」→ 次の連問セットの先頭に移動
- 「前の問題」→ 前の連問セットの先頭に移動
- 問325（最後の連問）→ 問326（スタンドアロン開始）
- 問326以降は従来通り1問ずつ移動

**境界条件:**
- セッションフィルタ（科目絞り込みなど）で連問の一部だけが選ばれた場合 → セット内の全問を自動的に含める（連問は分割不可）
- PracticePageから連問セットの途中の問題をタップした場合 → セット全体を表示し、タップした問題の位置まで自動スクロール（`currentQuestionId` を活用）

**実装箇所:** `QuestionPage.tsx` のナビゲーションロジック + セッション生成ロジック

#### 2-3. PracticePage の表示

- 連問はセット単位で1行表示（例: 「問196-199 PGE1製剤」）
- タップでセットごとQuestionPageを開く
- スタンドアロン問題（326-345）は従来通り1問1行

#### 2-4. useLinkedQuestions フックの簡素化

修復後は全問に `linked_group` が設定されるため:
- 隣接問題からの推定ロジック（パターン2）を削除
- `linked_group` の有無だけで判定するシンプルな実装に

#### 2-5. LinkedQuestionViewer の state 管理

- `group.groupId` を React の `key` として渡し、セット切り替え時に state を確実にリセット
- ディープリンクや途中問題アクセス時に古い回答状態が残らないようにする

### 3. テスト計画

#### データ品質テスト（修復スクリプト実行後）
- 全年度の問196-325が漏れなく `linked_group` に含まれることを検証
- 各グループの問題数が2-4の範囲であることを検証
- `linked_scenario` が空でないことを検証
- 各問の `question_text` に共通シナリオが重複していないことを検証

#### UIテスト
- 連問セットの先頭問題にアクセス → 全問が表示されること
- 連問セットの途中の問題にアクセス → セット全体が表示されること
- ナビゲーションがセット単位で動くこと
- スタンドアロン問題（326-345）が従来通り動くこと

#### 実機確認
- iPhone実機で連問セットを3つ以上確認
- シナリオ全文が読めること
- 各問の選択肢が問題文の直後にあること

### 4. スコープ外（今回やらないこと）

- 解説の修正（「親問題参照」→ 個別解説に差し替え）は別タスク
- 問326-345のデータ改善は対象外

### 5. 安全対策

- 修復スクリプトに `--dry-run` モードを実装（差分をログ出力、ファイル書き換えしない）
- dry-run で全年度の変更内容を確認してから本実行
- git commit 前にビルド成功を確認
