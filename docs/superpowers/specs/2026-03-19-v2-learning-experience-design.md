# pharma-exam-ai V2: 学習体験設計スペック

## Context

薬剤師国家試験対策アプリ「国試ノート」のV2設計。現在はMVP Phase 1-2が完了し、11年分3,795問+解説が搭載済み。しかし、問題を解くだけのツールに留まっており、「何をやるべきか」「どこが弱いか」の導線がない。

調査レポート（学習理論・ゲーミフィケーション・UI/UX・AI活用）を踏まえ、**学習体験の強化**を中心にV2全体像を設計する。

### ユーザーの実体験から得た洞察

- 基礎知識が不足していた受験生タイプ（模試で合格点に差があった）
- **科目ごとに弱点を重点的に潰す**学習スタイル
- 正答率60-70%以上の「落としてはいけない問題」に絞って学習
- 問題→知識不足発見→**付箋にまとめ→一問一答フラッシュカード**で反復
- 「理解」ではなく「**問題を解くために使える**」レベルまで反復
- 覚えた問題は飛ばす（**削除法**）
- **完璧を求めない** — 消去法で解ければOK
- クイズレットで混合型カード使用（用語↔定義、問い↔答え、語呂↔対象）

### 薬剤師国家試験の構造

- 345問中、正答率60%以上の「落とせない問題」が225〜260問
- 難問を解くより**基本問題のカバー率を上げる**のが合格戦略
- **出題基準**が厚労省から公式に公開（科目→大項目→中項目→小項目→例示）
- 薬ゼミ等の予備校は出題基準に過去問をマッピングして分析済み

## コアコンセプト

**「隙間時間を合格力に変えるコーチ」**

このアプリだけで国試勉強は完結しない。だからこそ、隙間時間（5-15分）を最大限活用して、基礎知識の定着と弱点潰しを習慣化する。

---

## フェーズ別KPI設計

ユーザーの学習フェーズでKPIが変わる：

| 時期 | 目的 | メインKPI | アプリの役割 |
|------|------|----------|------------|
| **前期（4-9月）** | 勉強習慣＋基礎固め | 学習継続日数・日の学習時間 | 暗記カード中心＋必須問題。「今日も5分だけやろう」 |
| **後期（9-2月）** | 合格点到達＋弱点潰し | 単元マスター率（中項目レベル） | 弱点科目の問題演習中心。「物理あと2単元」 |

---

## 学習コアループ

```
① 弱点発見 — 必須問題（落としてはいけない問題）を解く → 知識不足を発見
② 知識カード化 — 解説から暗記カードを生成（混合型）
③ 反復暗記 — 隙間時間にカードを回す。完璧じゃなくていい
④ マスター判定 — 同単元の類題3問以上で正答率80%+ → マスター認定
⑤ 削除法 — マスター済み単元は出題頻度↓ → 次の弱点にリソース集中
```

## データ構造：出題基準ツリー

厚労省の公式出題基準PDFをマスタデータとして取り込む。

```
科目（9科目：物理・化学・生物・衛生・薬理・薬剤・病態・法規・実務）
  └ 大項目（例：健康と健康）← グルーピング用
    └ 中項目（例：栄養素）← ★ マスター判定の単位
      └ 小項目（例：食事摂取基準）← ドリルダウンで見える
        └ 問題（過去問を小項目にマッピング）
        └ 暗記カード（問題の解説から生成、小項目に紐付く）
```

### マスター判定ロジック

- **マスター条件**: その中項目に紐づく「落とせない問題」の類題を3問以上解いて正答率80%+
- **学習中**: 1問以上解いているが正答率80%未満
- **未着手**: まだ1問も解いていない

## 画面構成（4タブ + サブ画面）

### ボトムナビゲーション

| タブ | 役割 | 主な要素 |
|------|------|---------|
| 🏠 ホーム | 進捗俯瞰 + 今日のメニュー | フェーズ別KPI / 科目別進捗バー / コーチ提案（時間適応） / クイックアクセス |
| 📝 問題 | 問題演習の入口 | 科目→中項目→小項目ドリルダウン / 年度別 / 必須・理論・実践 / 「落とせない問題」フィルタ |
| 🃏 カード | 暗記カードの管理と復習 | 今日の復習分 / デッキ（単元別） / カード作成 / 分散学習スケジュール |
| 📊 分析 | 詳細な進捗分析 | 単元マスターマップ（タイル） / ヒートマップ / 足切りライン表示 |

### サブ画面

- 問題演習画面（回答→正誤判定→解説）
- 解説画面（→カード生成ボタン）
- カード復習画面（フリック操作）
- 付箋管理
- 設定（フェーズ切替など）

### ホーム画面設計

**ハイブリッド型（ダッシュボード + コーチ）**

上部：
- メインKPI（前期: 継続日数＋学習時間 / 後期: 単元マスター率 18/47）
- サブ統計（解いた問題数・復習カード数・今週マスター数）
- 科目別マスター進捗バー

下部：
- 🎯 今日のメニュー（コーチ提案、時間に応じて切替）
  - 「5分だけ」→ カード12枚
  - 「15分ある」→ 必須問題10問
- クイックアクセス（自分で選ぶ・付箋・ランダム10）

### 暗記カード設計

**混合型フォーマット**（科目や内容に応じて形式を変える）：
- 用語 → 定義（例：CYP3A4 → 肝臓の主要な薬物代謝酵素）
- 問い → 答え（例：ワルファリンの作用機序は？ → ビタミンK依存性凝固因子の合成阻害）
- 語呂合わせ → 対象（例：ふじこちゃんは不死身 → フジコ→不整脈治療薬のクラス分類）

### 単元マスターマップ（分析タブ）

Khan Academy風のタイルマップ。中項目が1タイルで色分け表示：
- 🟢 マスター（正答率80%+、3問以上）
- 🟡 学習中（1問以上解いたが80%未満）
- 🔴 要強化（正答率が特に低い）
- ⬛ 未着手

## 設計上の重要判断

- ✅ **完璧を求めない設計** — 暗記カードは「だいたい覚えた→消去法で解ける」がゴール
- ✅ **時間適応メニュー** — 5分→カード、15分→問題演習。コーチが自動提案
- ✅ **出題基準ベース** — 厚労省の公式出題基準を骨格に。中項目でマスター判定
- ✅ **削除法の自動化** — マスター済み単元は出題頻度が自動で下がる
- ⚠️ **正答率データ** — 公式データなし。ユーザー演習データから蓄積して判定
